Theme Switcher

One store update re-renders every subscribed component simultaneously.

Reactive state A store is a pub/sub container. Any component can subscribe and will re-render when the store changes. Zero dependencies SWC has no runtime dependencies — not even on this theme switcher. It runs on native browser APIs only. Scoped styles Shadow DOM means these card styles cannot leak out, and page-level styles cannot bleed in. No class name collisions. Shared stores Import the same store in as many components as you need. They all stay in sync with zero extra wiring.
