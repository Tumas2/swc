Combining SWC with non-SWC code

The button above is not an SWC component — it's a plain HTML button wired up by a regular <script> tag below. It simulates what a WebSocket handler, analytics script, or third-party widget might do: update SWC state from outside the component tree.

This works because components/index.js calls exposeGlobally({ getStore, getAllStores }), which assigns those functions to window.SWC. Any script on the page can then call window.SWC.getStore('notificationStore') to read or update state — and every subscribed component re-renders automatically.

Open DevTools and run window.SWC.getAllStores() to inspect all registered stores.
